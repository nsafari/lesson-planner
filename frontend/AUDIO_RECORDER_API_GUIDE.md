# 🎵 راهنمای استفاده از ضبط‌کننده صوت با API

## 📋 مرور کلی

این سیستم ضبط صوت به دانش‌آموزان امکان می‌دهد تا:
1. ابتدا صوت نمونه را 3 بار گوش دهند
2. سپس صوت خود را ضبط کنند
3. صوت ضبط شده را به سرور ارسال کنند

## 🔧 نحوه استفاده

### 1. **تنظیمات اولیه**
- فایل `sound.html` را در مرورگر باز کنید
- در بخش "تنظیمات برای تست":
  - شناسه دانش‌آموز را وارد کنید (پیش‌فرض: 1)
  - شناسه تکلیف را وارد کنید (پیش‌فرض: 1)
  - روی "بروزرسانی تنظیمات" کلیک کنید

### 2. **مراحل ضبط صوت**
1. **گوش دادن به نمونه:** روی دکمه ▶ کلیک کنید تا صوت نمونه پخش شود
2. **تکرار:** این کار را 3 بار انجام دهید تا دکمه ضبط فعال شود
3. **ضبط:** روی دکمه ⚫ کلیک کنید تا ضبط شروع شود
4. **توقف:** دوباره روی دکمه کلیک کنید تا ضبط متوقف شود
5. **شنیدن:** روی دکمه ▶ کلیک کنید تا صوت ضبط شده را بشنوید
6. **ارسال:** روی دکمه 📤 کلیک کنید تا صوت به سرور ارسال شود

## 🌐 API Integration

### **Endpoint ارسال:**
```
POST /students/:studentId/assignments/:assignmentId/submit
```

### **پارامترهای ارسالی:**
- **فایل صوتی:** `audioFile` (FormData)
- **نمره روزانه:** `dailyScore` (پیش‌فرض: 85)
- **وضعیت تکمیل:** `isCompleted` (پیش‌فرض: true)
- **یادداشت:** `notes` (پیش‌فرض: "تکلیف صوتی ارسال شد")
- **زمان صرف‌شده:** `timeSpent` (محاسبه خودکار)
- **وضعیت:** `status` (پیش‌فرض: "submitted")

### **مثال کد JavaScript:**
```javascript
// تنظیم شناسه‌ها
window.setStudentId(1);
window.setAssignmentId(5);

// ارسال صوت (اتوماتیک هنگام کلیک روی دکمه ارسال)
// فایل صوتی به سرور ارسال می‌شود
```

## 📁 ساختار فایل‌ها

### **فایل‌های اصلی:**
- `frontend/users/sound.html` - صفحه اصلی ضبط صوت
- `frontend/assets/js/sound.js` - منطق ضبط و ارسال صوت
- `frontend/assets/js/api/api.service.js` - سرویس ارتباط با API

### **وابستگی‌ها:**
- `recorder.js` - کتابخانه ضبط صوت
- `sweetalert2` - نمایش پیام‌ها
- `tippy.js` - راهنمای دکمه‌ها

## 🚀 تست API

### **1. راه‌اندازی سرور:**
```bash
cd backend
npm start
```

### **2. باز کردن صفحه ضبط صوت:**
```
frontend/users/sound.html
```

### **3. تنظیم شناسه‌ها:**
- دانش‌آموز: 1
- تکلیف: 1

### **4. ضبط و ارسال صوت:**
- صوت نمونه را 3 بار گوش دهید
- صوت خود را ضبط کنید
- روی دکمه ارسال کلیک کنید

## 📊 بررسی نتیجه

### **در سرور:**
- فایل صوتی در `backend/public/uploads/submissions/` ذخیره می‌شود
- رکورد در جدول `assignment_submissions` ایجاد می‌شود
- فیلد `audioFileUrl` آدرس فایل را نگه می‌دارد

### **در فرانت‌اند:**
- پیام موفقیت نمایش داده می‌شود
- رابط کاربری ریست می‌شود
- فایل صوتی پاک می‌شود

## ⚠️ نکات مهم

1. **احراز هویت:** API نیاز به توکن معتبر دارد
2. **فرمت فایل:** فایل‌های MP3, WAV, OGG پشتیبانی می‌شوند
3. **حجم فایل:** حداکثر 10 مگابایت
4. **دسترسی میکروفون:** کاربر باید اجازه دسترسی به میکروفون بدهد
5. **ES6 Modules:** فایل‌ها از ES6 modules استفاده می‌کنند
